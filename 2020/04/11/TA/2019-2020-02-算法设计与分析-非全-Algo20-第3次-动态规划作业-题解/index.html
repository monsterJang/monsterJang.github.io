7-1 整数拆分

思路

注意到n,k不超出100，可以对所有情况进行预处理

令 dp[i][j] 表示将整数 n 无序拆分成最大数为 k 的拆分数

则对于当前的 dp[i][j]，先计算最大数为 j 且包含 j 的方案数，有 dp[i][j] = dp[i - j][min(j, i - j)]

对于当前的 i，计算完所有的 j 之后，对其进行一个累加，即可得到将 i 无序拆分成最大数为 j 的拆分数，即 dp[i][j] += dp[i][j - 1]

7-2 最大子段和

思路

注意题目要求算法的时间复杂度为O(n)

令 dp[i] 表示 1~i 中以 i 这个位置为结尾的子段可以得到的最大子段和（以 i 这个位置为结尾的子段意味着 i 这个位置上的数必须用到）

则有 dp[i] = max(dp[i - 1], 0) + a[i];

最后遍历一遍所有的 dp[i] 取最大值即可

当然，也可以不用 dp[] 数组而用一个变量直接记录这个过程

7-4 整数n拆成平方数的和

思路

01背包问题的变型，参考：

  • 背包九讲——全篇详细理解与代码实现
  • 背包九讲.pdf

给定的代码按照背包问题的描述写出，参照如下：

  • 给定的正整数 n => 背包的容量 V
  • N个正整数 x => N个物品，得到的价值 w[i] = x，耗费的背包容量 c[i] = x * x，注意 N=sqrt(V)
  • dp[i][j] => 用前 i 个正整数耗费背包容量为 j 时能得到的最大价值
  • 这道题即为：有 N 件物品和一个容量为 V 的背包。放入第 i 件物品耗费的费用是 c[i] ，得到的价值是 w[i]。求解将哪些物品装入背包可使价值总和最大，且满足“恰好装满背包”和“输出字典序最小的最优方案”

注意到这道题要求输出最小字典序的方案，所以对物品编号（即对应的正整数）做了 x <== n + 1 - x 的变换

7-5 凑零钱

思路

思考一下可以发现，这道题和7-4 整数n拆成平方数的和的原理是一样的

01背包问题的变型，参考：

  • 背包九讲——全篇详细理解与代码实现
  • 背包九讲.pdf

给定的代码按照背包问题的描述写出，参照如下：

  • 要付的款额 m => 背包的容量 V
  • n个硬币 => N个物品，得到的价值 w[i] = 1，表示用了一个硬币；耗费的背包容量 c[i]，表示硬币的面值
  • dp[i][j] => 用前 i 个硬币耗费背包容量为 j 时能得到的最大价值
  • 这道题即为：有 N 件物品和一个容量为 V 的背包。放入第 i 件物品耗费的费用是 c[i] ，得到的价值是 w[i]。求解将哪些物品装入背包可使价值总和最大，且满足“恰好装满背包”和“输出字典序最小的最优方案”

注意到这道题要求输出最小字典序的方案，所以对物品做了按 c[i] 降序排序的处理
