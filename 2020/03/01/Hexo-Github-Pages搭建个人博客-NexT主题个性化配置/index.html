使用Hexo + Github Pages搭建个人博客已经越来越简单，网上的教程也有很多，这里贴一个我搭建时主要参考的链接：超详细Hexo+Github Page搭建技术博客教程【持续更新】，以及主要记录一下自己的NexT主题的相关配置。

站点配置文件：/_config.yml。 位于站点根目录下，主要包含 Hexo 本身的配置

主题配置文件：/themes/next/_config.yml。位于主题目录下，这份配置由主题作者提供，主要用于配置主题相关的选项

选择Scheme

Scheme 是 NexT 提供的一种特性，借助于 Scheme，NexT 为你提供多种不同的外观。同时，几乎所有的配置都可以 在 Scheme 之间共用。目前 NexT 支持的Scheme有： Muse | Mist | Pisces | Gemini

打开主题配置文件，找到scheme字段，更改为想要的值，这里选择了Pisces

设置菜单

菜单配置包括两个部分，第一是菜单项（名称、链接和对应的图标），第二是菜单项的显示文本。 NexT 使用的是 Font Awesome 提供的图标。

设置菜单项和菜单项对应的链接和图标

编辑主题配置文件，找到menu字段，格式为：key: /link/ || icon

在创建完菜单项之后，对应于每个菜单项的链接需要生成对应页面。

per_page 为 true，表示只渲染那些在 Front-matter 中具有mathjax：true的页面。如：

此外，要注意的是，使用Katex引擎渲染，在输入公式时有一些限制。

  • 显示的数学公式（即 $$ ... $$ ）在开始的 $$ 前方和结束的 $$ 后方（comment #32），不得有任何字符（空格除外）。
  • 不支持 Unicode（comment #32）。
  • 内联公式 (..$..$) 在开始的 $ 之后和结尾的 $ 之前必须没有空格
  • 如果在正文标题中使用数学公式（即 ## Heading ），那么在相应的目录项中，它将显示3次相关的 LaTex 代码
  • 如果在文章标题中使用数学公式，则它将不会被呈现。
